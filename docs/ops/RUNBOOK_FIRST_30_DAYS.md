# RUNBOOK_FIRST_30_DAYS.md — OpenClaw Post-Launch Operations

This defines how the system should behave during the first 30 days.

The goal is stability and credibility, not speed.

---

## Week 1 — Establish trust

Allowed:

- max 3 posts/day globally
- max 1 site active
- max 1 niche active

Focus:

- consistent posting cadence
- verifying indexing
- verifying analytics ingestion

DO NOT:

- launch new sites
- change templates frequently
- flood with content

---

## Week 2 — Validate content patterns

Review:

- impressions
- crawl stats
- indexing delays
- internal link structure

If indexing slow:

- reduce posting frequency
- increase internal linking depth

---

## Week 3 — Expand topic clusters

Allowed:

- expand content within same niche
- create supporting articles
- strengthen topical authority

Still avoid:

- launching multiple niches

---

## Week 4 — Evaluate first scaling decision

Check:

- which posts gained impressions
- which keywords triggered visibility
- which content types worked best

Only then:

- consider adding second site
- consider adding second cluster

---

## Reality check

If zero traffic in first month:

This is normal.

SEO is delayed.

The system is building crawl + trust + authority.

Do not panic-scale.
