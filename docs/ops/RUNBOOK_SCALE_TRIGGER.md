# RUNBOOK_SCALE_TRIGGER.md

OpenClaw Affiliate Automation System

Purpose: Defines EXACT signals that determine:

- when to increase publishing volume
- when to launch new sites
- when to expand topic clusters
- when to hold steady
- when to STOP scaling entirely

Scaling too early destroys ranking potential.
Scaling too late wastes growth momentum.

---

## Core Rule

We NEVER scale based on:

- number of articles published
- how good content "looks"
- how much effort was invested

We scale ONLY on measurable signals.

---

## Signal Tiers

### TIER 1 — INDEXING SIGNALS (Earliest stage)

These appear first.

Check:

- pages indexed within 24–96 hours
- sitemap crawled regularly
- Search Console shows impressions

If:

```
>70% of new pages indexed within 4 days
```

Site passes Tier 1.

### TIER 2 — IMPRESSION SIGNALS

Now check:

- impressions rising week over week
- long-tail queries appearing
- pages showing in positions 15–50

If:

```
total impressions increasing weekly
```

Site passes Tier 2.

### TIER 3 — EARLY TRAFFIC SIGNALS

Check:

- organic clicks starting
- at least some pages < position 20

If:

```
any page reaches top 20 consistently
```

Site passes Tier 3.

### TIER 4 — CONVERSION SIGNALS

Check:

- affiliate clicks measurable
- referral traffic recorded
- engagement healthy

If:

```
any affiliate click tracked organically
```

Site passes Tier 4.

---

## Scaling Actions

### If Tier 1 ONLY

Do NOT add new sites.

Action:

- expand existing cluster slowly

### If Tier 2 reached

Allowed:

- increase article frequency slightly
- expand same niche cluster

Still: NO new site launches yet.

### If Tier 3 reached

Allowed:

- launch second niche cluster
- increase publishing rate

Still avoid mass scaling.

### If Tier 4 reached

SAFE TO:

- add new domains
- deploy additional sites
- increase automation throughput

**This is the real green light.**

---

## STOP SCALING CONDITIONS

Immediately freeze growth if:

- indexing suddenly drops
- impressions collapse
- hosting uptime unstable
- publishing errors spike
- queue backlog grows endlessly

**Fix root cause first.**

---

## Final Rule

Scaling without signals is gambling.

**Scaling with signals is engineering.**
