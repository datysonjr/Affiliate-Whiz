# TOPIC_DISCOVERY_ENGINE.md

OpenClaw Affiliate Automation System

Purpose: Defines how OpenClaw continuously discovers new profitable topics and niches without relying on manual brainstorming.

The system must find:

- buyer-intent search queries
- scalable topic clusters
- weakly defended SERPs
- niches that allow affiliate monetization

OpenClaw does NOT chase trends. It discovers sustainable revenue ecosystems.

---

## Core Discovery Philosophy

We are NOT searching for:

- highest volume keywords
- viral trends
- popular niches

We search for:

**repeatable buyer questions inside product ecosystems**

Revenue comes from decision questions, not popularity.

---

## Discovery Pipeline

OpenClaw should treat topic discovery as a loop:

```
existing niche → extract user questions → expand variations →
detect weak coverage → validate monetization → queue for publishing
```

---

## Stage 1 — Seed Extraction

From any existing article or cluster:

Extract:

- product names
- use cases
- buyer concerns
- comparisons mentioned
- price ranges referenced

Each becomes a potential topic generator.

Example:

```
office chair article
  → chair for back pain
  → chair under $300
  → chair for tall people
  → chair vs gaming chair
```

---

## Stage 2 — Question Expansion

For each seed:

Generate:

- "best X for Y"
- "X vs Z"
- "X alternatives"
- "how to choose X"
- "is X worth it"

These represent real search behavior.

---

## Stage 3 — Weak SERP Detection

OpenClaw must flag topics where:

- search results contain small blogs
- content outdated
- articles shallow
- affiliate pages thin

Reject topics dominated by:

- huge media brands
- government domains
- massive retail aggregators

---

## Stage 4 — Monetization Check

Before topic approved:

Must confirm:

- at least 3 purchasable products exist
- affiliate programs available
- user decision required

If no buying decision → reject topic.

---

## Stage 5 — Cluster Formation

Never publish single topics.

Each approved topic must generate:

```
1 main page + 5–20 support pages
```

Only clusters get queued.

---

## Discovery Frequency

Run topic discovery:

- weekly minimum
- after every 20 articles published
- after every revenue milestone

Never stop discovery loop.

---

## Final Rule

If OpenClaw stops discovering topics,

**growth stops permanently.**
