# COMPETITOR_WEAKNESS_SCANNER.md

OpenClaw Affiliate Automation System

Purpose: Defines how OpenClaw automatically detects weak competitor pages that can be outranked quickly.

SEO success is not about writing the best article.
It is about targeting the weakest existing results.

---

## Core Principle

Google rankings are not a merit system.

They are: **relative strength comparisons.**

We do not need perfect content.

We only need: **content stronger than the weakest top-ranking page.**

---

## Weak Page Detection Signals

OpenClaw should flag SERP targets where results show:

### 1. Thin Content

Indicators:

- short articles (<1000 words)
- minimal structured sections
- few headings
- shallow product analysis

Thin pages are easiest to beat.

### 2. Outdated Content

Indicators:

- year older than current
- old product models referenced
- broken links
- missing newer alternatives

Outdated affiliate guides are extremely common.

### 3. Poor Internal Linking

Indicators:

- article rarely linked internally
- no visible site cluster support
- orphan-like page

Weak link support = fragile ranking.

### 4. Weak Domain Authority Signals

Indicators:

- small niche blogs ranking
- hobbyist sites ranking
- minimal backlinks visible
- low editorial depth

These represent soft targets.

### 5. Bad UX Structure

Indicators:

- no comparison table
- no FAQ section
- cluttered layout
- excessive ads
- slow page load

Google increasingly penalizes these.

---

## Opportunity Scoring

Each SERP receives:

```
Thin Content Score
Outdated Score
Weak Domain Score
Structure Weakness Score
```

Combine into:

```
WEAKNESS_TOTAL
```

Target only SERPs with:

```
WEAKNESS_TOTAL >= threshold
```

---

## Attack Strategy

When a weak SERP found:

OpenClaw must:

1. build full cluster (not single page)
2. deploy superior article blueprint
3. interlink immediately
4. publish support pages within same week

Never attack with one isolated page.

**Clusters overwhelm weak competitors.**
