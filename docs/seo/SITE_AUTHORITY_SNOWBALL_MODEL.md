# SITE_AUTHORITY_SNOWBALL_MODEL.md

OpenClaw Affiliate Automation System

Purpose: Defines how a small affiliate site grows from:

10 pages → 50 pages → 200 pages → 500+ ranking pages.

Authority does NOT grow linearly.
It grows exponentially when clusters compound.

---

## Stage 1 — Seed Phase (0–20 pages)

Goal: **Prove topical relevance.**

Strategy:

- 1 core buyer guide
- 5 support guides
- 10 informational posts

Focus: **depth over volume.**

Publishing speed must remain slow.

---

## Stage 2 — Trust Phase (20–80 pages)

Goal: **Build topical dominance.**

Strategy:

- expand same cluster only
- publish scenario-based guides
- publish comparisons
- strengthen internal links

Never jump niches yet.

---

## Stage 3 — Expansion Phase (80–200 pages)

Goal: **Own the entire topic ecosystem.**

Strategy:

- answer EVERY buyer question
- cover ALL price tiers
- cover ALL use cases

At this stage: Google begins auto-indexing faster.

---

## Stage 4 — Authority Phase (200–500 pages)

Goal: **Become default reference site.**

Strategy:

- launch adjacent clusters
- link clusters together
- build navigation hubs

This creates authority snowball effect.

---

## Publishing Speed Rules

Never:

```
10 articles day one
```

Instead:

```
3/week → 5/week → 7/week → scale gradually
```

Sudden spikes trigger ranking suppression.

---

## Snowball Signals (Safe to Scale)

Scale ONLY when:

- pages indexing within 48–72h
- impressions rising steadily
- internal pages ranking for long-tail queries

If none present:

**Do NOT scale content.**

Fix cluster structure first.

---

## Final Rule

Traffic comes from:

**topic depth, not article count.**
