# MONEY_PAGE_PRIORITIZATION.md

OpenClaw Affiliate Automation System

Purpose: Defines how OpenClaw identifies and prioritizes high-revenue pages.

Not all pages are equal.

Some pages generate: **80% of revenue.**

These pages must receive:

- internal linking priority
- update priority
- homepage exposure
- faster refresh cycles

---

## Core Rule

Revenue pages must be treated as infrastructure.

They are NOT just content.

---

## Money Page Identification Criteria

A page qualifies if ANY:

1. Contains multiple affiliate links
2. Targets "best X" buyer queries
3. Receives affiliate click traffic
4. Appears in top 20 search positions
5. Has highest internal engagement

---

## Priority Scoring

Each page receives:

```
Affiliate Density Score
Ranking Position Score
Traffic Score
Click-through Score
```

Combine into:

```
MONEY_PRIORITY_SCORE
```

Top 20% highest scores = **MONEY PAGES.**

---

## Reinforcement Actions

For money pages:

### 1. Internal Link Boost

They must receive:

- at least 2x internal links vs normal pages

### 2. Homepage Exposure

Top money pages must:

- appear in navigation
- appear in featured guides
- receive sidebar exposure

### 3. Update Frequency

Money pages must be refreshed:

```
every 30–45 days
```

Normal pages:

```
every 90–120 days
```

### 4. Cluster Support Expansion

If a page converts:

OpenClaw must generate:

- comparison articles
- alternatives pages
- buyer mistake guides
- troubleshooting guides

All linking back.

---

## Anti-Dilution Rule

Never publish: **too many equal-priority pages.**

This splits ranking authority.

Instead: **Push winning pages harder.**

---

## Final Rule

**Scaling winners beats creating new losers.**
