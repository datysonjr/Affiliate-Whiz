# ARTICLE_BLUEPRINT_TEMPLATE.md

OpenClaw Affiliate Automation System

Purpose: Defines the exact article structure that maximizes:

- fast indexing
- strong topical authority signals
- high scroll depth
- affiliate link clicks
- user trust
- internal linking power

Every article OpenClaw generates MUST follow this structure unless explicitly overridden.

This template is optimized for:

- informational + commercial hybrid intent
- SEO-first affiliate ranking pages
- scalable multi-site deployment
- AI-friendly structured parsing

---

## The Core Rule (Read First)

OpenClaw articles are NOT blog posts.

They are: **structured conversion documents disguised as helpful guides**.

Every section exists for a reason. Nothing is filler.

---

## Master Article Structure

### 1. TITLE (SEO + Click Optimized)

Must include:

- primary keyword
- year (if relevant)
- intent phrase

Format Examples:

```
Best [PRODUCT CATEGORY] for [USE CASE] (2026 Guide)
[PRODUCT CATEGORY]: Complete Buyer's Guide + Top Picks
How to Choose the Right [PRODUCT] for [SCENARIO]
```

---

### 2. ABOVE-THE-FOLD QUICK ANSWER BLOCK (CRITICAL)

This is the money block. Appears immediately after intro.

Structure:

```
If you want the short answer:

Best overall → [Affiliate Product Link]
Best budget → [Affiliate Product Link]
Best premium → [Affiliate Product Link]

(Click any option to check current price)
```

WHY:

- captures impatient buyers immediately
- prevents bounce
- improves CTR massively

**THIS SECTION IS REQUIRED.**

---

### 3. TRUST-BUILDING INTRO (SHORT)

Length: 80–120 words

Purpose:

- confirm user problem
- promise clear answer
- signal guide authority

NOT:

- storytelling
- filler
- long background

---

### 4. BUYER DECISION SUMMARY TABLE

VERY IMPORTANT FOR CONVERSIONS.

Example:

| Product | Best For | Price Range | Quick Verdict |
|---|---|---|---|
| Product A | Beginners | $$ | Most balanced option |
| Product B | Professionals | $$$ | Highest performance |
| Product C | Budget users | $ | Best cheap pick |

Include affiliate links in product names.

---

### 5. PRODUCT SECTION BLOCKS (CORE REVENUE)

Each product section must follow identical structure:

```markdown
### Product Name (Affiliate Linked)

**Best For:** [user type]

Short 2–3 sentence explanation.

**Pros**
- bullet
- bullet
- bullet

**Cons**
- bullet
- bullet

**Why We Recommend It**

Short explanation focused on:
- real-world usability
- reliability
- buyer confidence

**Check Price / Availability**

[Check latest price here →](affiliate-link)
```

MANDATORY: No more than 5–7 product blocks per article.
Too many options reduce conversions.

---

### 6. HOW TO CHOOSE SECTION (SEO GOLD)

This is NOT about products.

This section:

- ranks for informational queries
- builds topical authority
- reduces thin-affiliate penalties

Example subsections:

```
What features actually matter
Common mistakes buyers make
When you should NOT buy this product
```

This section drives long-tail traffic.

---

### 7. REAL-WORLD SCENARIO MATCHING

Example:

```
Best for beginners
Best for small budgets
Best for heavy use
Best for occasional use
```

Each scenario references product again.
This reinforces affiliate links naturally.

---

### 8. FAQ BLOCK (VERY IMPORTANT)

Google LOVES this.

Minimum 4 questions.

Example:

```
Is [product category] worth it?
What should I avoid when buying?
How long do these usually last?
Can beginners use them?
```

Each answer: 50–120 words.

---

### 9. FINAL RECOMMENDATION BLOCK (SECOND MONEY SHOT)

This repeats the top picks again.

Structure:

```
If you want the safest choice → [affiliate]
If you want best performance → [affiliate]
If you want cheapest good option → [affiliate]
```

---

### 10. INTERNAL LINK SECTION

MANDATORY. At bottom:

```
Related guides:
[internal link]
[internal link]
[internal link]
```

This powers site-wide ranking growth.

---

## Strict Rules OpenClaw Must Follow

- No fake testing claims
- No "we personally tested" unless true
- No fluff paragraphs
- No overuse of affiliate links
- No keyword stuffing

---

## Ideal Article Length

- minimum: 1,200 words
- target: 1,800–2,500 words
- max: 3,000 words

Shorter pages rarely rank.

---

## Conversion Psychology Principles

Every article must:

1. answer quickly
2. reduce decision anxiety
3. narrow choices
4. repeat top picks multiple times
5. never overwhelm

Too many options = lost sales.

---

## OpenClaw Implementation Notes

Agents must:

1. detect primary keyword intent
2. select 3–5 products max
3. populate template sections
4. enforce identical structure
5. insert affiliate links programmatically
6. auto-generate FAQ from keyword variations

---

## Final Rule

If an article does NOT include:

- quick answer block
- comparison table
- structured product blocks
- FAQ
- final recommendation

It is **INVALID** and must be regenerated.
